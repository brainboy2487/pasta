## Language and runtime foundation  
These steps ensure Pasta can express and execute the loops and math Mandelbrot needs.

### 1. Verify lexer stability  
- Confirm the three lexer fixes are present:  
  - `..` emits `DotDot`  
  - `"STEP"` maps to `TokenType::Step`  
  - duplicate `'/'` arm removed  
- Run `cargo build --release`  
- Run a small lex test: `FOR i IN 0..10 STEP 2:` should tokenize cleanly.

### 2. Add a `range(start, end)` builtin  
- Add a new `Value::Range { start, end, current }` variant.  
- Add a builtin function `range(a, b)` that returns this value.  
- Add a simple iterator method in the interpreter to advance `current` until `end`.  
- Add unit tests:  
  - `range(0,3)` ‚Üí yields `0,1,2`  
  - `range(2,2)` ‚Üí yields nothing  
  - `range(5,3)` ‚Üí yields nothing (or error, depending on your design)

### 3. Extend the interpreter‚Äôs `ForIn` handler  
- Detect when the iterable is a `Value::Range`.  
- Loop from `start` to `end`, assigning the loop variable each iteration.  
- Ensure loop variable is scoped correctly (local to the loop body).  
- Add tests:  
  - `FOR i IN range(0,3): PRINT i` prints `0 1 2`.

### 4. Confirm numeric expression support

## Graphics subsystem  
These steps turn Pasta into a language capable of drawing pixels in a native window.

### 5. Implement the X11 backend  
- Fill in `XOpenDisplay`, `XCreateSimpleWindow`, `XCreateGC`.  
- Implement `XPutImage` using the canvas byte buffer.  
- Implement event polling with `XPending` and `XNextEvent`.  
- Add WM_DELETE_WINDOW handling so closing the window sets `open = false`.  
- Add a Rust smoke test that:  
  - creates a window,  
  - draws a gradient into a canvas,  
  - calls `blit`,  
  - keeps the window open until closed.

### 6. Implement graphics builtins  
- In `graphics/builtins.rs`, wire each builtin to your runtime:  
  - `WINDOW(title, w, h)`  
  - `CANVAS(w, h)`  
  - `PIXEL(canvas, x, y, r, g, b)`  
  - `BLIT(window, canvas)`  
  - `WINDOW_OPEN(window)`  
  - `CLOSE(window)`  
- Add tests for canvas pixel writes and window creation.

### 7. Register graphics builtins  
- Call `graphics::register_builtins(rt)` during runtime initialization, or  
- Call it from your import handler when `FROM: <graphics>` appears.  
- Add a test script that draws a single red pixel and displays it.
